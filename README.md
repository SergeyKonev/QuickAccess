# Quick Access

**Версия:** 1.3.0

Расширение для браузера (Chrome / Edge / Firefox) для быстрой навигации по Bitrix24-порталам и автоматизации рутинных задач.

## Установка

1. Клонируйте репозиторий
2. Откройте `chrome://extensions/` (или `edge://extensions/`)
3. Включите **Режим разработчика**
4. Нажмите **Загрузить распакованное расширение** и выберите папку проекта

Расширение готово к работе сразу после установки. Настройки заполняются через вкладку **Settings** в popup.

## Функции

### Закладки

Быстрые ссылки на часто используемые страницы портала.

- **Добавить текущую страницу** (`+`) — сохраняет URL текущей вкладки
- **Создать вручную** (`+` с ссылкой) — модалка с полями "Название" и "/путь"
- **Открыть** — клик по закладке открывает страницу в новой вкладке рядом с текущей
- **Редактировать имя** — клик по названию закладки
- **Перетаскивание** — drag & drop через ручку `⋮⋮` для изменения порядка
- **Удалить** — кнопка `x`

Кнопка **Контроллер** открывает контроллер портала в режиме инкогнито (работает только для тестовых порталов). URL контроллера настраивается в Settings.

### Сниппеты

Сохранение и выполнение PHP-кода на текущем портале.

- **Выполнить код** (`>`) — одноразовое выполнение PHP без сохранения
- **Создать сниппет** — сохранить код для повторного использования
- **Запуск сниппета** — клик по `>` или по названию, открывает результат в новой вкладке
- **Редактирование** и **перетаскивание** — аналогично закладкам

Код выполняется по адресу `{портал}{путь_выполнения_PHP}?PHPCode=y&CODE=...`
Путь по умолчанию: `/admin/debug.php` (настраивается в Settings).

**Ctrl+Enter** в поле кода — выполнить/сохранить.

### Опции (Options)

Чтение и установка опций `Bitrix\Main\Config\Option` без захода в админку.

- **Получить** — читает текущее значение опции по модулю и названию
- **Установить** — записывает новое значение
- **Пресеты** — сохранение часто используемых опций для быстрого доступа. Клик по пресету подставляет значения в форму

### Kibana

Быстрый доступ к логам Kibana с фильтром по текущему порталу.

- Автоматически определяет RU или COM по домену портала
- Через выпадающее меню (`v`) можно выбрать конкретную версию вручную
- URL-шаблоны Kibana настраиваются в Settings (плейсхолдер `{portalUrl}`)

### Тарифы

- **Активация тарифа** — выбор купона из списка и активация через `BX.ajax.runAction`. После активации нужно обновить страницу через `Ctrl+F5`
- **Подписка маркета** — установка/снятие подписки через API. Требует заполненных network-credentials в Settings

Купоны добавляются в Settings -> Купоны лицензий.

### Настройки (Settings)

Все параметры расширения в одном месте:

| Параметр | Назначение |
|----------|------------|
| URL контроллера | Шаблон URL, плейсхолдер `{portalUri}` |
| Путь выполнения PHP | Эндпоинт для сниппетов (по умолчанию `/admin/debug.php`) |
| Kibana RU / COM URL | Шаблоны URL, плейсхолдер `{portalUrl}` |
| URL подписки маркета | Эндпоинт API подписок |
| Email / Пароль нетворка | Credentials для API маркета |
| Купоны лицензий | Список пар "Название — Купон" |

Настройки хранятся в `chrome.storage.local` и загружаются автоматически.

### Экспорт / Импорт

Резервное копирование и перенос данных между пользователями.

**Экспорт:**
- Чекбоксы для выбора: настройки, закладки, сниппеты
- Скачивается JSON-файл `quick-access-backup-YYYY-MM-DD.json`
- Email и пароль нетворка **не экспортируются** (безопасность)

**Импорт:**
- Загрузка JSON-файла
- Поддерживает текущий формат (v2.0) и старые форматы (только закладки / только сниппеты)
- Существующие credentials сохраняются при импорте
- Если credentials отсутствуют — появится напоминание заполнить их в Settings

### Mantis Version Checker

Контент-скрипт для страниц багрепортов Mantis (`jabber.bx`).

- Добавляет кнопку **Версия на эталоне** на форму баг-репорта
- Автоматически подставляет версию модуля с эталонного портала в поле версии

## Горячие клавиши

| Клавиша | Контекст | Действие |
|---------|----------|----------|
| `Ctrl+Enter` | Поле кода сниппета | Выполнить / сохранить |
| `Enter` | Поле имени закладки/сниппета | Подтвердить |
| `Esc` | Любая модалка | Закрыть |

## Структура проекта

```
popup/               — UI расширения
  controllers/       — бизнес-логика (модалки, импорт/экспорт, тарифы, настройки)
  features/          — фичи (сниппеты, опции, кибана, контроллер)
  services/          — сервисы (хранилище закладок, browser API, уведомления)
  views/             — компоненты отображения
background/          — Service Worker (API маркета, запросы к эталону)
content-scripts/     — скрипты для веб-страниц (Mantis)
shared/              — общие модули (схема настроек, загрузчик, утилиты)
```
